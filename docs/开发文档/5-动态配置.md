# 动态配置

## 目标

将硬编码的 Provider / Model / API Key 抽出为可持久化的本地配置，通过设置面板运行时修改并重启 opencode，不再需要改代码。

## 存储方案

`~/.config/tdbot/config.json`：

```json
{
  "provider": "deepseek",
  "model": "deepseek-v4-flash",
  "apiKey": "sk-xxx",
  "baseUrl": ""
}
```

### configStore.ts

```typescript
export interface AppConfig {
  provider: string
  model: string
  apiKey: string
  baseUrl: string      // 自定义 Provider 的 API 地址
}

loadConfig()  → 读取或返回默认值
saveConfig(c) → 写回 JSON
```

## 重启 opencode 机制

```typescript
async function restartOpencode(config: AppConfig) {
  if (opencodeServer) opencodeServer.close()
  opencodeReady = false

  const { createOpencode } = await import('@opencode-ai/sdk/v2')
  const { client, server } = await createOpencode({
    config: {
      model: `${config.provider}/${config.model}`,
      agent: { general: { prompt: '你是一个名为 tdbot 的桌面智能体助手。' } },
      provider: {
        [config.provider]: {
          options: {
            apiKey: config.apiKey,
            ...(config.baseUrl ? { baseURL: config.baseUrl } : {}),
          },
          models: { [config.model]: { id: config.model } },
        },
      },
    },
  })
  opencodeClient = client
  opencodeServer = server
  opencodeReady = true
}
```

内置 Provider（deepseek / anthropic / openai）使用 opencode 的默认 API 地址，`baseUrl` 为空。自定义时通过 `baseURL` 传入地址。

## IPC 协议

| 方法 | 参数 | 返回 | 说明 |
|------|------|------|------|
| `get-config` | — | `AppConfig` | 读取当前配置 |
| `save-config` | `AppConfig` | `{ success }` | 保存并重启 opencode |
