# 响应内容优化

> **状态：核心展示功能已保留，实现方式有变更**
> - `parseCodeBlocks()` + `markdown.ts` → **已替换**：改用 `react-markdown` + `remark-gfm` 组件化渲染（见 `MessageItem.tsx`）
> - `ScrollArea` Viewport ref → **已替换**：改用原生 `<div>` + `useAutoScroll` hook
> - 代码高亮依然使用 `highlight.js`，集成在 `MessageItem.tsx` 的 `CodeBlock` 组件内

## 目标

优化 AI 回复内容的展示体验，包括代码块语法高亮、复制按钮、加载动画、思考过程折叠等。

---

## 1. 代码块语法高亮 + 复制按钮

### 技术选型

| 项目 | 选择 | 理由 |
|------|------|------|
| 高亮引擎 | **highlight.js** | 轻量，支持 197 种语言，无需额外配置 |
| 主题 | `github` | 与 Shadcn/ui 风格一致，浅色干净 |
| CSS 引入 | `globals.css` 中 `@import` | 零配置 |

### 安装

```bash
npm install highlight.js
```

### 当前实现

`CodeBlock` 组件定义在 `MessageItem.tsx` 内，由 `react-markdown` 的 `code` 组件覆写调度：

| 场景 | 渲染组件 | 说明 |
|------|----------|------|
| 有语言标识（`\`\`\`js`） | `CodeBlock` | highlight.js 高亮 + 可复制 + 换行切换 |
| 无语言但多行（`\`\`\`\n...`） | `CodeBlock` | 纯文本展示，`whitespace-pre-wrap` 换行 |
| 内联代码（`` `code` ``） | inline `<code>` | 灰色背景圆角标签 |

```tsx
// MessageItem.tsx — code 组件覆写
code({ className, children, ...props }) {
  const lang = className?.replace('language-', '')
  const code = String(children).replace(/\n$/, '')
  if (lang) return <CodeBlock language={lang} code={code} />
  if (code.includes('\n')) return <CodeBlock code={code} />
  return <code className="text-sm bg-muted/50 px-1 rounded" {...props}>{children}</code>
}
```

### CodeBlock 组件结构

```
┌────────────────────────────────────────┐
│ python                    📋  ← hover  │
│────────────────────────────────────────│
│ def hello():                           │
│     print("你好")       ← 高亮显示     │
│────────────────────────────────────────│
│                                 ↩ ↔   │
└────────────────────────────────────────┘
```

- 顶部栏：语言标签 + 复制按钮（点击显示"已复制"2 秒）
- 代码区域：`whitespace-pre-wrap break-all`（默认换行）/ `whitespace-pre`（不换行），通过 `↩`/`↔` 按钮切换
- 高亮：`hljs.highlight(code, { language })` 或 `hljs.highlightAuto(code)` 兜底

## 5. 消息时间戳

每条消息右下角显示格式化时间：

### 格式化规则

```
今天消息       → 14:30
昨天消息       → 昨天 14:30
今年内的消息   → 6/4 14:30
更早的消息     → 2025/6/4 14:30
```

### 实现

`src/renderer/src/lib/formatTime.ts`：

```typescript
export function formatTime(isoString: string): string {
  const date = new Date(isoString)
  const now = new Date()
  const today = new Date(now.getFullYear(), now.getMonth(), now.getDate())
  const target = new Date(date.getFullYear(), date.getMonth(), date.getDate())
  const diffDays = Math.floor((today.getTime() - target.getTime()) / 86400000)

  const time = date.toLocaleTimeString('zh-CN', { hour: '2-digit', minute: '2-digit' })

  if (diffDays === 0) return time
  if (diffDays === 1) return `昨天 ${time}`
  if (date.getFullYear() === now.getFullYear()) {
    return `${date.getMonth() + 1}/${date.getDate()} ${time}`
  }
  return `${date.getFullYear()}/${date.getMonth() + 1}/${date.getDate()} ${time}`
}
```

### 数据流

1. `useChat.sendMessage()` 创建消息时设置 `createdAt = new Date().toISOString()`
2. 流式更新保持 `createdAt` 不变
3. `saveToStorage()` 将 `createdAt` 持久化到 `sessions.json`
4. `loadMessages()` 从存储加载时携带 `createdAt`
5. `MessageItem` 调用 `formatTime()` 渲染
