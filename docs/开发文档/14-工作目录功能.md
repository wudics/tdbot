# 工作目录功能

## 目标

支持多工作目录隔离，每个工作目录包含独立的会话数据、配置、Skills 和 MCP 设置。切换工作目录时数据完全隔离，互不干扰。

## 工作目录结构

```
workspace-name/
├── tdbot.db              ← SQLite（会话 + agents + MCP + skills 开关）
├── config.json           ← 配置（provider/theme/skillToggles）
├── skills/               ← Skills 文件
├── output/               ← AI 可写入的目录
└── .opencode-data/       ← opencode 自身数据（可选，当前未启用）
```

## 切换方式

### 命令行

```bash
npm run dev -- --cwd=/path/to/workspace
npm start -- --cwd=/path/to/workspace
```

### 默认路径

未指定 `--cwd` 时，按运行模式确定根目录：

```typescript
function getAppRoot(): string {
  if (app.isPackaged) {
    // AppImage: APPIMAGE 环境变量，Windows exe: exe 路径
    const exePath = process.env.APPIMAGE || app.getPath('exe')
    return path.dirname(exePath)
  }
  // 开发模式使用项目根
  return path.join(__dirname, '..', '..')
}

function resolveWorkspace(): string {
  const arg = process.argv.find(a => a.startsWith('--cwd='))
  if (arg) return path.resolve(arg.split('=')[1])
  // 读取上次保存的工作目录
  const storePath = getWorkspaceStorePath()
  try {
    const saved = readFileSync(storePath, 'utf-8').trim()
    if (saved && existsSync(saved)) return saved
  } catch {}
  const appRoot = getAppRoot()
  const defaultWs = path.join(appRoot, 'workspace')
  if (!existsSync(defaultWs)) mkdirSync(defaultWs, { recursive: true })
  return defaultWs
}
```

**模式**：

| 模式 | 默认工作目录 |
|------|-------------|
| `npm run dev` | `{项目根}/workspace/` |
| AppImage | `{AppImage 所在目录}/workspace/` |
| Windows exe | `{exe 所在目录}/workspace/` |
| 指定 `--cwd` | 任意路径 |

### UI 切换

侧边栏底部显示当前工作目录路径，右侧有「切换」按钮：

```
┌──────────┬─────────────────────┐
│ 侧边栏    │ 聊天区               │
│          │                     │
│ ├ 对话1  │                     │
│ ├ 对话2  │                     │
│          │                     │
│ [+ 新建] │                     │
│──────────│                     │
│ 📁 /path │                     │
│   /ws    [切换] ← 点击弹出选择器  │
└──────────┴─────────────────────┘
```

点击「切换」后弹出系统目录选择器，选择新目录后自动重启 opencode。

## 工作目录持久化

切换后的工作目录路径保存在 `~/.config/tdbot/workspace-path.txt` 中，下次启动自动恢复。


## 数据隔离

| 数据 | 位置 | 是否跟随 workspace |
|------|------|:----------------:|
| `tdbot.db`（会话 + agents + MCP） | `{workspace}/` | ✅ |
| `config.json`（provider/theme 等） | `{workspace}/` | ✅ |
| `skills/` | `{workspace}/` | ✅ |
| `opencode.db`（opencode 自身数据） | `~/.local/share/opencode/` | ❌ 共享（sessionID 天然隔离） |
| `auth.json`（API Key） | `~/.local/share/opencode/` | ❌ 共享 |

切换工作目录时，`opencode.db` 中的 session 按 ID 隔离，不同 workspace 的会话不会混淆。

## 实现原理

### 启动流程

```
app.whenReady
  → resolveWorkspace()  ← 解析 --cwd 或读取保存路径或使用默认
  → saveWorkspacePath() ← 保存工作目录路径
  → process.chdir(workspacePath) ← 切换工作目录
  → 注册全部 IPC handler      ← 确保渲染进程可用
  → initDatabase()          ← 在 workspace 下创建 tdbot.db
  → createWindow()           ← 创建窗口
  → restartOpencode(config)  ← 启动 opencode
```

### 切换流程

```
用户点击 [切换] 按钮
  → setWorkspacePath IPC
  → dialog.showOpenDialog() 选择目录
  → saveWorkspacePath() + chdir
  → initDatabase() + restartOpencode()
  → 返回前端刷新会话列表 + toast 提示
```

## 友好错误提示

API 错误时显示中文友好提示而非原始 JSON：

| 错误类型 | 提示 |
|---------|------|
| 401 / 403 Authorization | API Key 未配置或无效，请在设置中检查 Provider 的 API Key 配置 |
| 400 Content Exists Risk | 请求被内容安全策略拦截，请换个方式描述问题 |
| 429 rate limit / quota | 请求频率过高或额度已用完，请稍后再试 |
| 其他 | 显示原始错误信息 |

通过 `friendlyError()` 函数实现：

```typescript
function friendlyError(err: any): string {
  const data = err?.data || err
  const msg = (data?.message || data?.error || '').toString()
  const status = data?.statusCode || 0

  if (status === 401 || status === 403 || msg.includes('Authorization') || ...) {
    return 'API Key 未配置或无效，请在设置中检查 Provider 的 API Key 配置'
  }
  // ...
}
```

## Toast 提示

切换工作目录后底部居中显示绿色 toast：

```
工作目录已切换
```

- 桌面端：在聊天区中央（`md:left-[calc(50%+100px)]`，补偿侧边栏宽度）
- 移动端：全屏居中（`left-1/2`）
- 绿底白字，3 秒后自动消失
- 从下往上的滑入动画，`_forwards` 保持最终位置不偏移

```css
@keyframes toast-in {
  from { opacity: 0; transform: translate(-50%, 10px); }
  to { opacity: 1; transform: translate(-50%, 0); }
}
```
