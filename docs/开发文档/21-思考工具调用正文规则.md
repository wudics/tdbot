# 思考 / 工具调用 / 正文输出规则

## 目标

定义 tdbot 中模型响应消息的显示规则，覆盖推理文本、工具调用、正文三部分的排列、滚动行为和工具调用信息展示。

---

## 一、总体布局

每条 assistant 消息的结构：

```
┌─ 消息块 ──────────────────────────────────┐
│                                            │
│  ▼ 思考过程（可折叠标题行）                  │
│  ┌─ 思考区域 ──────────────────────────┐   │
│  │  [推理文本...按到达顺序]              │   │
│  │  [工具调用条目1 — 带关键参数]        │   │
│  │  [推理文本继续...]                   │   │
│  │  [工具调用条目2 — websearch 结果]    │   │
│  │  [工具调用条目3 — bash 执行]         │   │
│  └──────────────────────────────────────┘   │
│                                            │
│  ┌─ 正文区域 ──────────────────────────┐   │
│  │  ReactMarkdown 渲染正文             │   │
│  └──────────────────────────────────────┘   │
└─────────────────────────────────────────────┘
```

### 规则

1. **思考 + 工具调用在同一个可折叠区域，按时间顺序从上往下排列**
   - 推理文本以 `message.part.delta` + `partType === 'reasoning'` 流持续追加
   - 工具调用在推理过程中穿插出现，按 arrival order 排列
   - 两者在同一个 scrollable 容器内

2. **正文区域独立在下方**
   - 只有 `partType === 'text'`（非 reasoning 和 tool）的内容进入正文
   - `ReactMarkdown` 渲染，与思考区域逻辑分离

3. **流结束时思考区域自动折叠**
   - 正在流式传输时展开（`isStreaming` 为 true）
   - 流结束后自动折叠回标题行"▶ 思考过程"

4. **代码块渲染规则**
   - 有语言标识（`\`\`\`js`）→ `CodeBlock` 组件（highlight.js 高亮 + 可复制 + 自动换行切换）
   - 无语言标识但多行（`\n`）→ `CodeBlock` 组件（纯文本展示，`whitespace-pre-wrap` 正确换行）
   - 无语言标识且单行（`` `内联代码` ``）→ inline `<code>` 样式
   - 多行判定条件：`code.includes('\n')`

5. **消息底部 agent 名称标签**
   - assistant 消息在时间戳左侧显示 agent 名称，格式：`bg-muted` 标签 + `·` + 时间
   - 数据来源：每条消息创建时记录当时的 `activeAgent` ID，持久化到 DB `messages.agent` 列
   - 查找名称：从 `agents[]` 列表中匹配 `id`，找到则显示 `name`（如"执行模式"），未找到回退显示原始 ID（如 `build`）
   - user 消息不显示 agent 标签
   - 旧消息（`agent` 列为 NULL）不显示标签

## 二、智能滚动

### 当前问题

切换会话时自动滚动到底部尚未实现（`useAutoScroll` 只在用户靠近底部时触发，会话切换不触发）。

### 改进方案

新增 `useAutoScroll` hook（通用逻辑，适用于主 ScrollArea 和思考区域独立滚动容器）：

```typescript
function isNearBottom(el: HTMLElement, threshold = 50): boolean {
  return el.scrollHeight - el.scrollTop - el.clientHeight < threshold
}
```

| 用户位置 | 行为 |
|---|---|
| 底部附近（< 50px） | 自动滚动到最新内容 |
| 已往上滚动（>= 50px） | 不滚动，什么都不做 |
| 手动滚回到底部附近 | 恢复自动滚动 |

无悬浮按钮，无额外 UI 元素。

### 适用范围

- **主消息容器**（`ChatView.tsx`，原生 `<div>` 替代了 Radix ScrollArea）：正文区域 + 所有消息的整体滚动
- **思考区域**（`MessageItem.tsx` 内每个消息的思考容器）：独立管理，互不影响

### 文件改动

| 文件 | 改动 |
|---|---|
| `src/renderer/src/hooks/useAutoScroll.ts` | 新增，通用 hook |
| `src/renderer/src/components/ChatView.tsx` | 替换无条件 scroll effect，使用 `useAutoScroll`；Radix ScrollArea → 原生 `<div>` |
| `src/renderer/src/components/MessageItem.tsx` | 思考区域 div 添加 `useAutoScroll` |

## 三、工具调用显示改进

### 当前状态

工具条目只显示：`⏳ {toolName} — {title}`，没有输入参数、没有进度、多次调用同一工具用 name + pending 匹配会出错。

### 改进内容

#### 1. 关键参数始终可见

从 `part.state.input` 中根据 tool 名提取关键字段：

| 工具名 | 显示格式 |
|---|---|
| `web_search` / `websearch` / `search` | `🌐 搜索"{query}"` |
| `read` / `file_read` | `📖 读取文件 {filePath}` |
| `bash` / `execute_command` / `terminal` | `💻 {command.slice(0,60)}...` |
| `edit` / `write` / `file_edit` | `✏️ 编辑文件 {filePath}` |
| `glob` | `🔍 搜索模式 {pattern}` |
| `grep` / `search` | `🔍 {pattern} in {path}` |
| `task` / `subtask` | `📋 {description}` |
| 其他 | 只显示工具名 |

#### 2. 生命周期渲染

由 SDK v2 `message.part.updated` 事件驱动（统一通道 `stream-tool`，携带全部状态）：

| `part.state.status` | 对应阶段 | UI 表现 |
|---------------------|----------|---------|
| `running` | 工具开始执行 | 创建条目，显示 `⏳` + input 关键参数；如有 `progress.content` 则实时显示在条目下方 |
| `completed` | 工具执行成功 | `✅`，output 可折叠展开（websearch 走专用渲染，其他工具走 `<pre>`） |
| `error` | 工具执行失败 | `❌`，error 可折叠展开 |

- 运行中的工具：`⏳` 静态图标
- 有 progress content 时：实时显示在条目下方（浅灰色小字）
- 完成有 output 时：显示"展开结果"按钮，点击展开 `<pre>` 代码块
- 失败有 error 时：不显示内联错误摘要（与成功一致），仅显示"展开错误"按钮，点击展开完整错误信息

#### 3. 工具匹配改用 callID

```typescript
interface ToolCall {
  callID: string            // 唯一标识，替代 tool name + pending 匹配
  tool: string
  status: 'pending' | 'running' | 'completed' | 'error'
  input: Record<string, any>
  output?: string
  title?: string
  error?: string
  progress?: {
    structured?: Record<string, any>
    content?: { type: 'text' | 'file'; text?: string; uri?: string }[]
  }
}
```

#### 4. Tool 条目展开状态

- 默认折叠 output/error
- 点击"展开结果"或"展开错误"按钮展开
- 点击"收起结果"或"收起错误"收拢
- 每个条目独立控制
- 条目布局：`flex-nowrap` 容器，工具名称过长时 `truncate` 截断，展开/收起按钮 `flex-shrink-0 whitespace-nowrap` 确保始终在同一行右侧

### 文件改动

| 文件 | 改动 |
|---|---|
| `src/main/index.ts` | 在 `message.part.updated` 事件中识别 `part.type === 'tool'`，转发 `callID`、`tool`、`status`、`input`、`output`、`title`、`error`、`progress` 到 renderer |
| `src/preload/index.ts` + `index.d.ts` | 定义 `stream-tool` IPC 通道的 `onTool` 监听方法 |
| `src/renderer/src/hooks/useChat.ts` | 用 `callID` 匹配替代 name + status 匹配；新增 progress 状态 |
| `src/renderer/src/components/MessageItem.tsx` | `toolLabelRich()` 显示 input 关键参数 + 结果数；`WebSearchResults` 专用渲染；progress content 实时显示；error 可折叠展开 |

## 四、websearch 专用渲染

### 数据来源

websearch 工具的输出 `t.output` 为纯文本格式，每个结果块以 `\n\n---\n\n` 分隔：

```
Title: 搜索结果标题
URL: https://example.com/article
Published: 2026-04-30T01:35:03.000Z
Author: 作者
Highlights:
正文内容首段...
```

### 解析方式

```typescript
function parseWebSearchResults(output: string): { title: string; url: string; highlights: string }[] {
  const blocks = output.split(/\n\n---\n\n/)
  return blocks.map(block => ({
    title: block.match(/^Title:\s*(.*)/m)?.[1] || '',
    url: block.match(/^URL:\s*(.*)/m)?.[1] || '',
    highlights: block.split(/^Highlights:\n/m)[1]?.trim() || '',
  })).filter(r => r.url)
}
```

### 渲染设计

**标题行（始终可见）：**

```
✅ 🌐 搜索"2026年AI趋势" — 找到 5 条结果  [▼/▲ 展开/收起]
```

**展开状态：**

```
🔗 2026最新广西旅游必去景点推荐...
  2026年广西纯玩旅游首选桂林漓江与崇左德天瀑布...
🔗 【廣西景點2026】廣西10大名勝推介！
  廣西位於中國南部，以其壯麗的喀斯特山水與濃厚的民族文化聞名...
```

每条结果：
- **🔗 标题**：蓝色可点击，调用 `window.api.openExternal(url)` 在系统浏览器打开
- **描述**：`Highlights` 第一段非空文本，`line-clamp-2` 限制两行
- **URL**：鼠标悬停 `title` 显示完整 URL

### 判定条件

`t.tool === 'websearch'`（工具名为 `websearch`，无下划线）

### 新增 IPC

| 方向 | 通道 | 用途 |
|------|------|------|
| renderer → main | `open-external` | `shell.openExternal(url)` 在系统默认浏览器打开链接（Electron 跨平台 API，Linux / Windows / macOS 均有效） |

### 文件改动

| 文件 | 改动 |
|------|------|
| `src/main/index.ts` | +3 行：`ipcMain.handle('open-external', ...)` |
| `src/preload/index.ts` | +1 行：`api.openExternal` 桥接 |
| `src/preload/index.d.ts` | +1 行：`openExternal` 类型声明 |

## 五、布局与容器变更

### 1. Radix ScrollArea → 原生 `<div>`

**原因：** Radix ScrollArea 的 Root 元素内部有 `display: table; min-width: 100%`（Radix 内部设置），导致宽度由内容决定（CSS table 布局算法）。当 Viewport 内有宽内容（markdown `<table>`、长代码块等）时，table 会被撑开，而 `overflow-x: hidden` 只裁剪视觉溢出、不改变元素计算宽度，造成布局撑开、标题栏出现水平滚动。

**修复：** 移除 Radix ScrollArea（及相关 import），使用原生 `<div>`：

```tsx
- <ScrollArea className="h-full border rounded-lg overflow-y-auto min-w-0" ref={scrollRef}>
+ <div className="h-full border rounded-lg overflow-y-auto min-w-0" ref={scrollRef}>
```

**优势：**
- 原生 `<div>` 宽度完全由 CSS 约束，内容再宽也不会撑开容器
- `useAutoScroll`、滚动按钮、`scrollTo`/`scrollHeight` 等所有功能完全兼容
- 构建体积略减（不再包含 Radix ScrollArea 的 JS/CSS）

### 2. 空状态汉堡按钮

**问题：** 无会话时的空状态页面（`!sessionId` early return）没有渲染汉堡菜单按钮，窄屏下无法打开侧栏切换/新建会话。

**修复：** 添加与主视图一致的 `md:hidden` 汉堡按钮：

```tsx
// 空状态页面结构
<div className="flex-1 flex flex-col min-h-0 relative">
  <div className="flex items-center p-2 md:hidden">
    <Button variant="ghost" size="sm" onClick={onOpenSidebar}>
      <Menu className="h-5 w-5" />
    </Button>
  </div>
  <div className="flex-1 flex flex-col items-center justify-center text-muted-foreground gap-4">
    <p>选择一个对话或创建新对话</p>
    <Button variant="default" onClick={onNewSession}>
      <Plus className="h-4 w-4 mr-2" />
      新建对话
    </Button>
  </div>
</div>
```

## 六、用户消息导航条

### 目标

在消息列表左侧提供垂直导航条，标记用户发送的消息位置，点击快速跳转。

### 布局

```
┌─ flex row ────────────────────────────────────┐
│  ┌─ nav strip ──┐  ┌─ scroll container ──────┐ │
│  │  w-6          │  │  flex-1, overflow-y     │ │
│  │  max-h-[40%]  │  │                         │ │
│  │  self-center  │  │  MessageItem (纯渲染)   │ │
│  │               │  │                         │ │
│  │  · 用户消息1  │  │                         │ │
│  │  · 用户消息2  │  │                         │ │
│  │  · 用户消息3  │  │                         │ │
│  └───────────────┘  └─────────────────────────┘ │
└──────────────────────────────────────────────────┘
```

### 设计要点

| 特性 | 实现 |
|------|------|
| **定位** | 独立 `<nav>` 在 scroll container 左侧，`gap-1` 拉开间距 |
| **高度** | `max-h-[40%]` + `self-center`，垂直居中，不占满父容器 |
| **圆点** | 只渲染 `role === 'user'` 的消息，`w-2 h-2 rounded-full` |
| **点击** | `scrollIntoView({ behavior: 'smooth', block: 'start' })` 滚动到对应消息 |
| **滚动** | `overflow-y-auto` + 隐藏滚动条（`[scrollbar-width:none]` + `[&::-webkit-scrollbar]:hidden`） |
| **拖拽** | `cursor-grab` / `active:cursor-grabbing`，`onMouseDown/Move/Up` 实现 grab-scroll |
| **Tooltip** | `fixed` 定位浮动于页面之上，不受 overflow 裁剪；白色背景、边框阴影、260px 截断 |

### Tooltip 实现

```tsx
// state
const [tooltip, setTooltip] = useState<{ text: string; x: number; y: number } | null>(null)

// 圆点
<span
  onMouseEnter={e => setTooltip({ text: msg.text, x: e.clientX + 8, y: e.clientY })}
  onMouseMove={e => setTooltip(t => t ? { ...t, x: e.clientX + 8, y: e.clientY } : null)}
  onMouseLeave={() => setTooltip(null)}
/>

// 浮动 tooltip
{tooltip && (
  <div
    className="fixed z-50 text-xs text-gray-900 bg-white border rounded-md px-2.5 py-1.5 shadow-md pointer-events-none max-w-[260px] truncate"
    style={{ left: tooltip.x, top: tooltip.y }}
  >
    {tooltip.text}
  </div>
)}
```

| `src/renderer/src/components/ChatView.tsx` | 新增 nav 条 + drag-scroll + tooltip 逻辑；消息渲染恢复为纯 `<MessageItem>` |

## 七、消息底部 agent 名称标签

### 目标

在 assistant 消息的时间戳左侧显示生成该消息所用的 agent 名称，方便区分不同 agent 的回复。

### 数据流

```
sendMessage(text, system, files, agentId):
  → 创建 userMsg / assistantMsg 时记录 agentId
  → saveToStorage 序列化 agent 字段
  → StoredMessage.agent → DB messages.agent 列
  → loadMessages 读取 agent → MessageItem 渲染
```

### 渲染

```tsx
// MessageItem.tsx
{!isUser && message.agentId && (
  <span className="inline-block bg-muted px-1 rounded mr-1">
    {agents?.find(a => a.id === message.agentId)?.name || message.agentId}
  </span>
)}
{formatTime(message.createdAt)}
```

- 仅 assistant 消息、仅在存在 `agentId` 时显示
- `agents` 列表查找名称，未找到回退显示原始 ID

### 文件改动

| 文件 | 改动 |
|------|------|
| `src/renderer/src/hooks/useChat.ts` | `Message` 接口新增 `agentId?: string`；`sendMessage` 接受第 4 参数 `agentId`；`saveToStorage` 映射 `agent` 字段 |
| `src/main/sessionStore.ts` | `StoredMessage` 新增 `agent?: string`；INSERT/SELECT 适配 |
| `src/main/db.ts` | `CREATE TABLE messages` 新增 `agent TEXT` 列 |
| `src/renderer/src/components/ChatView.tsx` | `handleSend` 传 `activeAgent` 给 `sendMessage`；`<MessageItem>` 传 `agents` prop |
| `src/renderer/src/components/MessageItem.tsx` | 接收 `agents` prop；时间戳左侧渲染 agent 名称标签 |

### 升级兼容

新装用户：建表时直接包含 `agent TEXT` 列。
旧库迁移：`ALTER TABLE messages ADD COLUMN agent TEXT`（`try/catch` 无声处理已存在的情况）。
