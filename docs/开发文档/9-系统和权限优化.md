# 系统和权限优化

## 权限配置

### 最终方案

```typescript
permission: {
  websearch: 'allow',
  bash: 'allow',               // shell 命令允许执行
  external_directory: 'deny',  // 禁止访问项目目录外文件
}
```

### 防护层次

| 防护层 | 效果 |
|--------|------|
| `external_directory: "deny"` | AI 无法访问 `/etc/`、`~/.ssh/` 等项目外文件 |
| prompt 约束 | "只能在 `output/` 目录下创建或修改文件，项目内其他文件只能读取" |
| 项目在 Git 管理 | 破坏性修改可回滚 |

### `bash: "ask"` 暂不可用

opencode 提供了两种权限回复 API：

| API | 请求路径 | 结果 |
|-----|---------|------|
| `permission.reply()` | `POST /permission/{requestID}/reply` | ✅ HTTP 200，❌ 不继续执行 |
| `permission.respond()` | `POST /session/{sessionID}/permissions/{permissionID}` | ✅ HTTP 200，❌ 不继续执行 |

两种方式都返回成功，但 opencode 服务器**不再发送后续事件**（仅有 `server.heartbeat`），工具执行无法恢复。可能原因是 opencode 的权限系统依赖 Web UI 的特定交互机制（如 SSE 事件 ID 追踪），无法通过独立 HTTP 调用正确恢复。待 opencode 后续版本修复后再启用。

## 端口冲突处理

### 动态端口分配

`restartOpencode` 中使用 `port: 0`，让 OS 自动分配空闲端口：

```typescript
const { client, server } = await createOpencode({
  port: 0,  // OS 自动分配可用端口
  config: { ... },
})
```

### 重启延迟

关闭旧 server 后等待 1 秒，确保端口释放：

```typescript
if (opencodeServer) {
  opencodeServer.close()
  await new Promise(resolve => setTimeout(resolve, 1000))
}
```

## SQLite 存储

### 迁移原因

原有的 JSON 文件存储（`sessions.json`）在会话和消息增多时读写效率下降，且无法按条件查询。

### 技术选型

| 项目 | 选择 | 理由 |
|------|------|------|
| 数据库 | **sql.js** | WebAssembly 实现，免 native 编译，跨平台 |

### 数据库文件

`~/.config/tdbot/tdbot.db`

### Schema

```sql
CREATE TABLE sessions (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  created_at TEXT NOT NULL
);

CREATE TABLE messages (
  id TEXT PRIMARY KEY,
  session_id TEXT NOT NULL,
  role TEXT NOT NULL,
  text TEXT NOT NULL,
  reasoning TEXT,
  tools TEXT,           -- JSON 序列化
  created_at TEXT NOT NULL,
  FOREIGN KEY (session_id) REFERENCES sessions(id) ON DELETE CASCADE
);
```

### 数据迁移

启动时检测旧 `sessions.json`，自动导入 SQLite 后删除 JSON 文件。迁移完成后代码已清理，当前 `db.ts` 仅为数据库初始化 + CRUD，约 35 行。

### 接口不变

`sessionStore.ts` 对外接口（`loadSessions` / `saveMessages` 等）完全不变，调用方零改动。

## 工具输出修复

`useChat.ts` 的 `onTool` handler 中，工具输出不再追加到 `message.text`，搜索内容仅存在于思考过程的「展开结果」中，不污染正文。

```typescript
// 改前
let newText = m.text
if (data.output) { newText += `\n\`\`\`\n${data.output}\n\`\`\`\n` }
return { ...m, tools, text: newText }

// 改后
return { ...m, tools }
```

## 连接状态监控 + 自动重连

### 检测机制

主进程每 10 秒调用 `client.global.health()` 检测 opencode 服务器状态：

```
┌─ 主进程 ─────────────────────────────────┐
│                                           │
│ 每 10 秒 → health() → ✅ → IPC → 🟢 已连接│
│                     → ❌ → IPC → 🔴 断开  │
│                     → 自动重连（最多 3 次）→ 🟡 重连中│
│                     → 3 次失败 → ⏸ 等待手动│
└───────────────────────────────────────────┘
```

### 状态指示

Header 标题左侧彩色圆点：

| 颜色 | 状态 | 说明 |
|------|------|------|
| 🟢 绿色 | `connected` | 已连接，正常 |
| 🟡 黄色（闪烁） | `reconnecting` | 自动重连中，最多 3 次 |
| 🔴 红色 | `disconnected` | 已断开，点击重连 |

### 主进程核心代码

```typescript
function startHealthCheck() {
  stopHealthCheck()
  reconnectAttempts = 0
  healthCheckInterval = setInterval(async () => {
    try {
      await opencodeClient.global.health()
      mainWindow?.webContents.send('connection-status', 'connected')
      reconnectAttempts = 0
    } catch {
      mainWindow?.webContents.send('connection-status', 'disconnected')
      stopHealthCheck()
      tryAutoReconnect()
    }
  }, 10000)
}

async function tryAutoReconnect() {
  while (reconnectAttempts < MAX_RECONNECT_ATTEMPTS) {
    reconnectAttempts++
    mainWindow?.webContents.send('connection-status', 'reconnecting')
    try {
      const config = loadConfig()
      await restartOpencode(config)
      startHealthCheck()
      return
    } catch {
      await new Promise(resolve => setTimeout(resolve, 3000))
    }
  }
  mainWindow?.webContents.send('connection-status', 'disconnected')
}
```

### UI 代码

```tsx
<span
  className={`inline-block w-2 h-2 rounded-full mr-2 flex-shrink-0 ${
    connectionStatus === 'connected' ? 'bg-green-500' :
    connectionStatus === 'reconnecting' ? 'bg-yellow-500 animate-pulse' :
    'bg-red-500 cursor-pointer hover:bg-red-400'
  }`}
  onClick={connectionStatus === 'disconnected' ? handleReconnect : undefined}
  title={connectionStatus === 'connected' ? '已连接' : connectionStatus === 'reconnecting' ? '重连中...' : '已断开，点击重连'}
```

## 多 Provider 配置

### 存储结构

`~/.config/tdbot/config.json`：

```json
{
  "providers": [
    { "name": "deepseek", "model": "deepseek-v4-flash", "apiKey": "sk-xxx", "baseUrl": "" },
    { "name": "anthropic", "model": "claude-sonnet-4-20250514", "apiKey": "sk-xxx", "baseUrl": "" }
  ],
  "activeProvider": "deepseek",
  "activeModel": "deepseek-v4-flash",
  "webSearch": false
}
```

### 实现原理

opencode 支持在同一个 config 中配置多个 Provider，`promptAsync` 支持按次指定 `model` 参数。因此启动时一次性注册所有 Provider，切换模型时只需改 `promptAsync` 的参数，**无需重启**。

### 启动注册

```typescript
const providerConfig: Record<string, any> = {}
for (const p of config.providers) {
  providerConfig[p.name] = {
    options: { apiKey: p.apiKey, ...(p.baseUrl ? { baseURL: p.baseUrl } : {}) },
    models: { [p.model]: { id: p.model } },
  }
}

const { client, server } = await createOpencode({
  config: {
    model: `${config.activeProvider}/${config.activeModel}`,
    provider: providerConfig,
  },
})
```

### 切换模型

```typescript
await opencodeClient.session.promptAsync({
  sessionID: sessionId,
  model: { providerID: config.activeProvider, modelID: config.activeModel },
  parts: [{ type: 'text', text }],
})
```
