# 多模态优化

## 问题背景

使用第三方 provider（如 SiliconFlow）的视觉模型时，通过 opencode 转发（toolcall=true）图片附件会失败，模型返回"不支持图片输入"，但直调 API（toolcall=false）可以正常解析图片。而 opencode Zen 模型（如 Mimo）在任何路径下都能正常处理图片。

## 根因分析

### 问题 1：重启后附件图片不显示

`messages` 表缺少 `files` 列，`saveMessages()` 从未持久化文件元数据，`loadMessages()` 也不返回 `files`。

### 问题 2：noReply 上下文注入失败

项目使用 `@opencode-ai/sdk/v2`（V2 SDK），但 noReply 注入代码使用了 V1 API 格式：

```ts
// ❌ V1 格式（被 V2 SDK 静默忽略）
await opencodeClient.session.prompt({
  path: { id: sessionId },
  body: {
    noReply: true,
    parts: [{ type: 'text', text: contextText }],
  },
})

// ✅ V2 格式
await opencodeClient.session.prompt({
  sessionID: sessionId,
  noReply: true,
  parts: [{ type: 'text', text: contextText }],
})
```

### 问题 3：第三方模型不支持图片附件

opencode 服务端根据注册的模型 `capabilities` 决定是否转发图片数据。自定义 provider 在注册时没有声明 `attachment` 和 `modalities`，导致 opencode 认为模型不支持图片输入，主动丢弃图片数据。

## 配置说明

### 模型能力配置字段（全部可选，按需填写）

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `attachment` | boolean | `false` | 文件附件（图片等） |
| `reasoning` | boolean | `true` | 思考过程展示 |
| `temperature` | boolean | `true` | 温度调节 |
| `tool_call` | boolean | `true` | 工具/函数调用 |
| `modalities.input` | string[] | `["text"]` | 输入模态：text, image, audio, video, pdf |
| `modalities.output` | string[] | `["text"]` | 输出模态 |
| `variants` | object | `{}` | 变体模式，如思考深度切换 |
| `limit.context` | number | - | 上下文窗口（tokens） |
| `limit.output` | number | - | 输出限制（tokens） |
| `options` | object | - | 透传给 API 的额外参数 |
| `headers` | object | - | 自定义请求头 |

### 示例：视觉模型配置

```json
{
  "attachment": true,
  "modalities": {
    "input": ["text", "image"],
    "output": ["text"]
  }
}
```

### 示例：DeepSeek thinking mode + variants

```json
{
  "reasoning": true,
  "variants": {
    "none": {},
    "high": { "options": { "thinking": { "type": "high" } } },
    "max": { "options": { "thinking": { "type": "max" } } }
  }
}
```

## 完整配置参考（SettingsDialog 中可展开查看）

```jsonc
{
  // === 核心能力 (设置 true 启用, false 关闭) ===
  "attachment": false,    // 文件附件 (图片等)
  "reasoning": true,     // 思考过程展示
  "temperature": true,    // 温度调节
  "tool_call": true,      // 工具/函数调用

  // === 输入输出模态 (不设则默认为仅文本) ===
  "modalities": {
    "input": ["text", "image", "audio", "video", "pdf"],
    "output": ["text"]
  },

  // === 变体模式 (如思考深度切换) ===
  "variants": {
    "none": {},
    "high": { "options": { "thinking": { "type": "high" } } }
  },

  // === 模型参数限制 ===
  "limit": {
    "context": 128000,    // 上下文窗口 (tokens)
    "output": 4096        // 输出限制 (tokens)
  },

  // === 额外参数 (透传给 API) ===
  "options": { "temperature": 0.7, "max_tokens": 4096 },

  // === 自定义请求头 ===
  "headers": { "X-Custom": "value" }
}
```

## 技术要点

- V2 SDK 的 `session.prompt()` 接受扁平参数 `{ sessionID, noReply, parts }`，不支持 V1 的 `{ path: { id }, body: { ... } }` 嵌套格式
- `modelCfg` 中所有字段都可选，未设字段使用 opencode 服务端默认值
- `modalities.input` 声明图片支持但不确定模型是否有视觉能力时，API 层面会返回错误（而非 opencode 静默丢弃）
- `attachment: true` 不等于 `modalities.input: ["image"]`，前者启用文件系统，后者声明图片输入能力
- `variants` 通过 `promptAsync({ variant })` 选择，opencode 将对应 variant 内容 merge 进请求配置

## loadAgents 合并逻辑

`loadAgents()` 始终合并内置 agents + 自定义 agents，与旧 DB 行为一致：

```ts
export function loadAgents(): AgentData[] {
  const config = loadConfig()
  const custom = config.agents || []
  const merged: AgentData[] = []
  for (const b of builtinAgents) {
    const override = custom.find(a => a.id === b.id)
    merged.push(override || b)
  }
  for (const a of custom) {
    if (!builtinAgents.some(b => b.id === a.id)) {
      merged.push(a)
    }
  }
  return merged
}
```

## handleSave 统一写入（SettingsDialog）

所有配置字段（providers、agents、mcps、theme、skillToggles 等）一次 `saveConfig` 写回，不再独立调用 `saveAgents()`/`saveMcp()`/`saveConfigField()`：

```ts
const handleSave = async () => {
  setSaving(true)
  const currentCfg = await window.api.getConfig()
  await window.api.saveConfig({
    ...currentCfg,
    ...config,
    agents,
    mcps,
    webInfo: currentCfg.webInfo,
  })
  setSaving(false)
  onOpenChange(false)
}
```

## builtin agent 权限禁用

AgentEditForm 中 Skills 权限和 MCP 权限的 `<select>` 增加 `disabled={form.builtin}`，与工具权限一致——builtin agent 只能编辑提示词，不能修改权限。

## send-message agent fallback

`promptAsync` 前校验 `activeAgent` 是否存在于 `loadAgents()` 返回的列表中，不存在则回退 `'build'`：

```ts
const allAgents = loadAgents()
const agentFallback = allAgents.some(a => a.id === savedConfig.activeAgent)
  ? savedConfig.activeAgent : 'build'

await opencodeClient.session.promptAsync({
  sessionID: sessionId,
  agent: agentFallback,
  ...
})
