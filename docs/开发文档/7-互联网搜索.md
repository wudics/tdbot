# 互联网搜索

## 目标

AI 回答问题时可以启用实时网络搜索。默认关闭，通过聊天工具栏开关控制，**即时生效，无需重启**。

## 实现原理

opencode 内置了 `websearch` 工具（基于 Exa AI），但需要满足两个条件：

1. **启动时设置环境变量** `OPENCODE_ENABLE_EXA=1`
2. **配置中设置权限** `permission: { websearch: "allow" }`

## 数据流

```
restartOpencode (始终):
  → process.env.OPENCODE_ENABLE_EXA = '1'
  → config.permission = { websearch: 'allow' }
  → config.tools = { web_search: true }
  → websearch 工具始终可用

切换开关（即时生效，无需重启）:
  工具栏 toggle → 保存 config.webSearch 到 config.json
  ↓
send-message 每次读取 config:
  → webSearch=true  → tools: { websearch: true }
  → webSearch=false → tools: { websearch: false }
  ↓
opencode 根据 tools 参数决定 AI 能否调用 websearch
```

## 环境变量

`restartOpencode` 中固定设置：

```typescript
process.env.OPENCODE_ENABLE_EXA = '1'
```

创建一个新的 opencode 服务器实例时，SDK 的 `server.ts` 会将 `...process.env` 传递到子进程，环境变量生效。

## 配置

### restartOpencode（始终启用 websearch）

```typescript
process.env.OPENCODE_ENABLE_EXA = '1'

const basePermission = {
  websearch: 'allow',
  webfetch: 'allow',
  read: 'allow',
  edit: 'allow',
  glob: 'allow',
  grep: 'allow',
  list: 'allow',
  external_directory: 'deny',
  ...mcpPermissions,   // MCP 工具权限
}

const { client, server } = await createOpencode({
  port: 0,  // OS 自动分配端口，避免冲突
  config: {
    permission: basePermission,
    agent: agentConfig,  // 各 agent 可 override 工具权限
    provider: providerConfig,
    // model, skills, mcp 等
  },
})
```

> 注意：`permission` 现在是全局基础权限，各 agent 通过 `toolPermissions` / `skillPermissions` / `mcpPermissions` 覆盖特定工具的 `ask` / `deny`。

### send-message（按开关控制）

```typescript
const tools = { websearch: savedConfig.webSearch }

await opencodeClient.session.promptAsync({
  sessionID: sessionId,
  agent: 'general',
  tools,  // true → 可用；false → 不可用
  parts: [{ type: 'text', text }],
})
```

## UI 设计

开关位于聊天框底部（消息区与输入框之间）：

```
┌──────────────────────────────────────┐
│ 消息区域                               │
│                                      │
├──────────────────────────────────────┤
│ 🌐 互联网搜索  [⬤——○]     ← 工具栏   │
├──────────────────────────────────────┤
│ [输入消息................] [发送]    │
└──────────────────────────────────────┘
```

Toggle 纯 CSS 实现，轨道 `h-4 w-8`，圆点 `h-4 w-4`，与底色等高重合。

### 开关状态

```tsx
<button onClick={toggleSearch}>
  🌐 互联网搜索
  <span className={`inline-block h-4 w-8 rounded-full transition-colors ${webSearch ? 'bg-primary' : 'bg-input'}`}>
    <span className={`block h-4 w-4 rounded-full bg-white transition-all ${webSearch ? 'ml-4' : 'ml-0'}`} />
  </span>
</button>
```

## 工具调用展示

AI 使用 websearch 时，思考过程中显示调用信息。当前采用专用渲染组件：

### websearch 专用展示

```
▶ 思考过程
  ⏳ websearch                           ← 调用中
  ✅ websearch — 找到 5 个结果           ← 完成后（含结果计数）
  ┌─────────────────────────────────────────┐
  │ Node.js — 官方文档                     │
  │ https://nodejs.org/                     │
  │ 最新的 Node.js 文档、API 参考和...     │ ← line-clamp-2
  │ ⋯                                      │
  │ ---                                    │
  │ Node.js — Wikipedia                    │
  │ https://en.wikipedia.org/wiki/Node.js  │
  │ ⋯                                      │
  └─────────────────────────────────────────┘
```

- `parseWebSearchResults(t.output)` 将文本输出解析为结构化结果列表
- 每个结果展示：标题（可点击 🔗 调用 `open-external` IPC 在浏览器打开）、URL、描述（`line-clamp-2`）
- `toolLabelRich` 函数识别 `websearch` 工具，显示 **结果数量** 后缀
- 结果区域内联展开，无需「展开结果」按钮

### 普通工具的回退展示（非 websearch）

```
  ✅ bash — curl https://api.example.com
    ┌────────────────────────────────────────┐
    │ { "status": "ok" }                    │ ← 代码块渲染
    └────────────────────────────────────────┘
```

## 端口冲突处理

`restartOpencode` 中关闭旧 server 后等待 1 秒释放端口，并使用 `port: 0` 让 OS 自动分配空闲端口，避免端口冲突。

## 内置工具

opencode 默认所有工具开启，无需额外配置：

| 工具 | 用途 |
|------|------|
| bash | 执行 shell 命令 |
| edit / write / patch | 文件修改 |
| read / grep / glob | 文件搜索 |
| webfetch | 获取网页内容 |
| **websearch** | **网络搜索（需 `ENABLE_EXA`）** |
| question | 向用户提问 |
| todowrite | 管理待办 |
| skill | 加载技能 |
